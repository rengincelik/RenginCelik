About Me

Most developers don't have engineering fundamentals.
Most engineers don't write production code.
I do both.

I'm a simulation and real-time systems developer with a B.Eng. from Istanbul Technical University — fluid mechanics, thermodynamics, heat & mass transfer, process systems design. That foundation is what separates my work: I don't implement physics equations by copying formulas, I understand what they model.

My main project, PipeFlow, is a browser-based real-time pipeline flow simulator built entirely from scratch — zero dependencies, no framework, no build step. It implements Darcy-Weisbach friction losses, Colebrook-White via Newton-Raphson iteration, pump H-Q curve fitting, and live operating point calculation. Everything renders in real time with SVG animation and live charts.

Before simulation, I spent three years in Unity — HLSL shaders, GPU instancing, URP pipelines, Editor tooling. Real-time systems and rendering are the common thread.

Projects

PipeFlow Simulator Featured

Vanilla JS · SVG · Zero Dependencies

Browser-based real-time pipeline flow simulator. Drag-and-drop pumps, pipes, valves, elbows and reducers — the physics engine recalculates pressure, velocity, flow rate and energy losses every 100ms. Results shown live via SVG animation, charts, and tooltips.

  • Darcy-Weisbach friction losses, Colebrook-White via Newton-Raphson (50 iter, tol 1×10⁻⁸)
  • Pump H-Q curve — 3-point quadratic fit; operating point via bisection each tick
  • Empirical fluid model — water and 50% ethylene glycol
  • Architecture: SimulationEngine · SVGRenderer · ChartRenderer · FlowAnimator — decoupled via EventEmitter
  • Zero dependencies. No framework. No build step.
